# 修復：陳本清視角下陳昱升位置錯誤

**日期**: 2026-04-04  
**問題**: 在陳本清視角下，陳昱升位置跑到最上面，應該在陳致真底下

## 根本原因分析

### 問題1：後端查詢邏輯不支持多家族
- 陳進陽 (b112) 的 `family_tree_id` = `family_陳_b112`（他的原生家族）
- 陳進陽的 `families` = `["chen_family", "family_陳_b112"]`（多家族歸屬）
- 後端 API 只使用 `family_tree_id` 查詢，所以陳進陽被排除
- 結果：陳昱升失去父親信息，無法正確構建樹結構

### 問題2：前端樹構建邏輯處理不當
- 即使陳進陽被包含在查詢中，由於他沒有父親/母親記錄 (parents: [])
- 他會被當作獨立的根節點
- 結果：陳昱升作為陳進陽的子女跑到最上面

## 修復方案

### 修復1：後端 API 查詢邏輯 (server.cjs, 行 1520-1526)

**變更前**:
```javascript
// 只查詢指定家族
sql = "SELECT * FROM members WHERE family_tree_id = ?";
params = [targetFamilyId];
```

**變更後**:
```javascript
// 使用 families JSON 字段進行多家族查詢
const familyLike = `%"${targetFamilyId}"%`;
sql = "SELECT * FROM members WHERE family_tree_id = ? OR families LIKE ?";
params = [targetFamilyId, familyLike];
```

**效果**: 現在陳進陽會被正確查詢到，因為他的 `families` 包含 `chen_family`

### 修復2：前端樹構建邏輯 (FamilyTree.jsx, 行 621-654)

**變更前**:
```javascript
// 沒有父母就直接成為根節點
if (!member.parents || member.parents.length === 0) return true;
```

**變更後**:
```javascript
// 檢查配偶是否有父母在家族內
if (!member.parents || member.parents.length === 0) {
    if (member.spouses && member.spouses.length > 0) {
        for (const spouseId of member.spouses) {
            const spouse = currentFamilyMembers.find(s => s.id === spouseId);
            if (spouse && spouse.parents && spouse.parents.length > 0) {
                const spouseHasParentInCurrentFamily = spouse.parents.some(pid => 
                    currentFamilyMembers.some(m => m.id === pid)
                );
                if (spouseHasParentInCurrentFamily) {
                    return false;  // 不作為根節點，跟隨配偶
                }
            }
        }
    }
    return true;  // 如果配偶也沒有父母，才作為根節點
}
```

**效果**: 陳進陽作為陳致真的配偶，會跟隨陳致真的樹而不是獨立成為根節點
